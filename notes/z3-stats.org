  When Axioms Get Instantiated

  Each counter is incremented inside a dedicated instantiate_* method. The trigger points are:

  1. Variable creation (mk_var) — if the term is a store and laziness() <= 1, axiom1 fires immediately.
  2. Select added as parent (add_parent_select) — axiom2a fires against all stores in the equivalence class; axiom2b fires unless array_delay_exp_axiom
  is set.
  3. Store added (add_store) — axiom2a fires against all parent selects of the array.
  4. Disequality (new_diseq_eh) — extensionality fires.
  5. Final check (final_check_eh) — delayed axiom2b instances are asserted; interface equalities are created (incrementing m_num_eq_splits).

  Deduplication via Fingerprinting

  Redundant instantiations are prevented using ctx.add_fingerprint() calls in theory_array_base.cpp. Each axiom type uses a unique fingerprint key — for
   example, store axiom2 fingerprints with the store owner ID and select arguments (line 229), and the extended axioms use dedicated constants like
  m_default_map_fingerprint = UINT_MAX - 112 (theory_array_full.h:41-45).
